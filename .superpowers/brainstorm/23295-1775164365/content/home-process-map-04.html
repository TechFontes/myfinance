Mapa de Processo — Como o projeto foi conduzido

Processo de engenharia

Spec → Plano → Execução

Metodologia

Cada feature começa com design spec, vira plano de implementação com tasks numeradas, e executa com TDD obrigatório. Nenhum código sem teste primeiro.

Bugs de produção → Recovery Program

Estabilização

Schema drift, dashboard stale, edit flows quebrados. Em vez de hotfix, criou-se um programa de recuperação com auditoria, 8 tasks priorizadas por wave e gates de verificação.

Execução paralela com agentes isolados

Coordenação

Tasks independentes rodaram em worktrees isolados. Category edit + dashboard freshness em paralelo. Seed + governance em paralelo. Integração com verificação cruzada.

Schema discipline como gate obrigatório

Governança

Migrations são lei. Testes verificam que toda mudança no schema tem migration correspondente. Deploy sem migration é bloqueante.

Publish
